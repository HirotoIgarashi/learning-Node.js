# 1. Node.jsとは何か

[Node.js]は2009年にRyan Dahlという開発者により作成されました。
[Node.js]はC++で記述された高速なJavaScriptインタプリタです。

[Node.js]では、低レベルなUNIX APIが使えるようになっています。

例えば低レベルなUNIX APIとは以下の通りです。

- プロセス
- ファイル
- ネットワークソケット
- HTTPクライアント
- サーバAPI

## [Node.js]のバージョンの説明

Node.jsのバージョンを１つ目のドットの後ろの数字が奇数なら開発版、偶数なら安定版であることを意味します。v0.1.xやv0.3xは開発版、v0.2xやv0.4xは安定版です。現在開発してる開発版はv0.9系、時期安定版はv1.0系になる予定です。

# 2. Node.jsのインストール

Node.jsのインストールには[nvm](Node Version Manager)を使用します。

[nvm]: https://github.com/nvm-sh/nvm

## 2.1. バージョンマネージャnvmのインストール

その時点のnvmの最新版を[nvm]のサイトで調べます。v.40.1の場合、以下のコマンドを実行します。

```fish
curl -o- https://raw.githubusercontent.com/nvm-sh/nvm/v0.40.1/install.sh | bash
```

バージョンを確認します。

```fish
nvm --version
0.40.1
```

## 2.2. nvmの使い方

### 2.2.1. 最新のActive LTSバージョンをインストール

```fish
nvm install --lts
```

### 2.2.2. 最新バージョンをインストール

```fish
nvm install node
```

### 2.2.3. インストール可能なバージョンの一覧を表示する

```fish
nvm ls-remote
```

### 2.2.4. バージョンを指定してインストール

```fish
nvm install 22.7.0
```

### 2.2.5. 使うバージョンを指定する

```fish
nvm use 22.7.0
```

### 2.2.6. インストール済みのバージョンの一覧を表示する

```fish
nvm use 22.7.0
```

### 2.2.7. インストール済みのバージョンを削除する

```fish
nvm uninstall 22.7.0
```

# 5. テスト環境の構築

## 5.1. ユニットテストツールの分類

- フレームワーク

テストコードの基本構造の記述、テストの設定、実行

- アサーション

それぞれのテストの中で、結果として得られた値が期待した値と一致するかどうかを検証する。

- テストダブル

テスト対象のコードが依存するモジュールやコンポーネントを任意の別のものに置き換える。

- カバレッジ

テスト対象のコード全体のうち、テストコードによってカバーされている部分の割合を解析する。
